# Telegram-бот RentSense

Бот отвечает за персональные уведомления о новых объявлениях. Пользователь задает фильтры, а бот периодически проверяет свежие записи в БД и отправляет подходящие варианты.

## Роль в проекте

RentSense состоит из нескольких контуров:

- `app/api` - единый backend и ML-инференс.
- `app/parser` и `app/scheduler` - сбор объявлений из внешнего источника.
- `app/ui` - Streamlit интерфейс.
- `app/bot` - Telegram-сценарий мониторинга.
- `ml` - подготовка данных, обучение моделей, SHAP-анализ.

Бот не обучает модель и не парсит сайт. Он использует БД для чтения объявлений и API для прогноза цены.

## Логика работы

1. Пользователь запускает бота и включает уведомления.
2. Пользователь задает фильтры: район, метро, комнаты, бюджет, площадь, время до метро.
3. Планировщик получает свежие объявления из БД.
4. Объявления фильтруются по настройкам пользователя.
5. Для подходящих объявлений запрашивается прогноз через API.
6. Вычисляется разница `predicted_price - offer_price`.
7. Пользователь получает лучший вариант с учетом лимитов и дедупликации.

## Компоненты модуля

- `telegram_bot.py` - команды, кнопки, callback handlers, запуск приложения.
- `database.py` - таблицы `bot_users`, `sent_alerts` и операции с ними.
- `scanner.py` - чтение свежих объявлений из MySQL.
- `alert_logic.py` - фильтрация, ранжирование и проверка лимитов.
- `scheduler.py` - периодическая рассылка и сброс дневных счетчиков.
- `predict_client.py` - HTTP-клиент для `/api/predict`.
- `templates.py` - форматирование сообщений.

## Команды

- `/start` - регистрация и главное меню.
- `/help` - список команд.
- `/on` - включить уведомления.
- `/off` - выключить уведомления.
- `/filters` - показать текущие фильтры.
- `/set key value` - установить фильтр текстовой командой.
- `/reset_filters` - сбросить фильтры.
- `/status` - статус подписки и дневного лимита.
- `/cancel` - отменить ввод значения.

## Фильтры

- `district`
- `metro`
- `rooms`
- `area_min`, `area_max`
- `price_min`, `price_max`
- `travel_time_max`

## Расписание и лимиты

- Рассылка работает каждый час с 9 до 23.
- Сброс дневных счетчиков выполняется в 00:00.
- За один запуск одному пользователю отправляется не более одного объявления.
- Повторная отправка одного `cian_id` одному пользователю блокируется через `sent_alerts`.

## Развертывание

В основном compose бот запускается сервисом `telegram_bot`. Для отдельного VPS используется `docker-compose.telegram-eu.yml`; в этом режиме бот подключается к удаленному backend через `API_BASE_URL`.
