# Telegram bot RentSense

Бот отправляет пользователю объявления, где цена в публикации ниже ожидаемой цены по модели.

## Как это работает

1. Пользователь настраивает фильтры в чате.
2. Планировщик получает свежие объявления за текущий день.
3. Для каждого объявления бот запрашивает прогноз у API.
4. Считается выгода: `predicted_price - offer_price`.
5. Пользователь получает лучшие подходящие объявления с учетом лимитов.

## Интеграции

- API: `app/api/main.py` (`/api/predict`)
- БД: таблицы `bot_users`, `sent_alerts`
- Сканер объявлений: `app/bot/scanner.py`
- Логика отбора: `app/bot/alert_logic.py`
- Планировщик отправки: `app/bot/scheduler.py`

## Команды

- `/start` - регистрация пользователя
- `/on` - включить уведомления
- `/off` - выключить уведомления
- `/filters` - показать фильтры
- `/set ключ значение` - установить фильтр
- `/reset_filters` - сбросить все фильтры
- `/status` - статус подписки и лимита
- `/cancel` - отменить ввод фильтра

Поддерживаемые фильтры:
- `district`
- `rooms`
- `area_min`, `area_max`
- `price_min`, `price_max`
- `metro`
- `travel_time_max`

## Ограничения и расписание

- Отправка идет в рабочие часы `9..23`.
- За один запуск пользователю отправляется не более одного объявления.
- Есть дневной лимит отправок.
- Повторная отправка одного `cian_id` одному пользователю блокируется через `sent_alerts`.

## Файлы модуля

- `telegram_bot.py` - команды, кнопки, callback обработчики.
- `scheduler.py` - запуск рассылки и сброс дневных счетчиков.
- `alert_logic.py` - отбор и ранжирование объявлений.
- `scanner.py` - чтение свежих объявлений из БД.
- `database.py` - модели и операции для таблиц бота.
- `templates.py` - формат сообщений.
